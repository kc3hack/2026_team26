# 疲労度測定システム（Fatigue Detection System）構成定義書

## 1. ソフトウェア概要
Webカメラとマイクからユーザーの生体情報を取得し、顔（目線・瞬き）および音声の解析を通じて「疲労度」を可視化。組織内での健康管理を支援するWindowsアプリケーションおよび管理Webプラットフォーム。

---

## 2. 技術選定 (Tech Stack)

### A. デスクトップクライアント (Windows)
* **メイン言語:** Java 21+ (JavaFX)
    * *選定理由:* WindowsデスクトップアプリとしてリッチなUI（MUI風デザイン）を構築でき、外部プロセス(Python)の制御に優れるため。
* **AI/解析エンジン:** Python 3.10+
    * **フェイストラッキング:** MediaPipe (軽量・高精度な目線・顔の向き抽出)
    * **疲労度ロジック:** OpenCV + Scipy (瞬き頻度: EAR算出, 眠気検知)
    * **音声解析:** librosa / PyAudio (声のトーン、ピッチ変化の抽出)
* **連携:** PyInstaller (Pythonをバイナリ化してJavaから呼び出し)

### B. バックエンド (Server)
* **ランタイム:** Node.js (TypeScript)
    * *選定理由:* JSONデータのやり取りがスムーズで、リアルタイム通知（Socket.io）への拡張性が高いため。
* **API形式:** REST API (Express)
* **データベース:** MySQL 8.0
    * *構成:* ユーザー情報、チーム所属、日次疲労度ログ、統計データの保存。

### C. フロントエンド (Web Dashboard)
* **フレームワーク:** React
* **UIライブラリ:** Material UI (MUI)
* **チャート表示:** Recharts または Chart.js (疲労推移のグラフ化)

---

## 3. 主要機能の要件定義

### ① デスクトップアプリ (Java & Python)
* **認証機能:** Node.jsサーバー経由のログイン・トークン保持。
* **リアルタイムモニタリング:**
    * カメラから目線の動き、瞬きの間隔(PERCLOS)を算出。
    * マイクから発話時の基本周波数の揺らぎを測定。
* **データ送信:** 5分〜10分おき、または異常検知時にサーバーへ「疲労スコア」を送信。
    * *注意:* プライバシー保護のため、画像そのものではなく「数値データ」のみを送信する。

### ② サーバー・DB (Node.js & MySQL)
* **ユーザー管理:** アカウント作成、JWT認証、パスワードハッシュ化。
* **グループ管理:** チーム作成、招待コードによる紐付け。
* **データ集計:** 個人別・チーム別の日次/週次疲労レポートの生成。

### ③ Webサイト (React)
* **管理者ダッシュボード:** チームメンバーの現在の疲労状況を一覧表示。
* **マイページ:** 過去の統計データの閲覧、疲労傾向の分析アドバイス表示。

---

## 4. 疲労測定のアルゴリズム（AI詳細）
1.  **目線・瞬き (Eye Aspect Ratio - EAR):**
    $EAR = \frac{||p_2 - p_6|| + ||p_3 - p_5||}{2||p_1 - p_4||}$
    この値が一定時間閾値を下回ると「眠気・疲労」と判定。
2.  **音声解析:** 平常時と比較して高域成分の減少や、発話速度の低下を検知。

---

## 5. 開発のステップ（推奨）
1.  **Phase 1:** Pythonで「カメラから瞬きと目線を数値化する」スクリプトを単体で完成させる。
2.  **Phase 2:** Java(JavaFX)からそのPythonを起動し、数値を読み取れるようにする。
3.  **Phase 3:** Node.js + MySQLでログインとデータ保存のAPIを作る。
4.  **Phase 4:** Reactで保存されたデータをグラフ表示する。
